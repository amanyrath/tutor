## Documentation Standards

When adding new documentation:

1. **Feature docs** go in `features/` - User-facing feature guides
2. **Implementation docs** go in `implementation/` - Technical implementation details
3. **Integration docs** go in `integrations/` - External service setup
4. **PR docs** go in `pr/` - Pull request implementation notes

All documentation should:
- Be clear and concise
- Include code examples where relevant
- Link to related documentation
- Be kept up to date with code changes
